# 회원 도메인 원칙

## 1. 도메인 목적
회원 도메인은 사용자의 계정 생성, 인증, 프로필 관리를 담당한다.

## 2. 핵심 원칙

### 인증 위임 원칙
- 인증(Authentication)은 Supabase Auth에 완전히 위임한다
- 비밀번호 저장, 토큰 발급/갱신 등 인증 로직을 직접 구현하지 않는다

### 최소 정보 원칙
- 서비스 제공에 필요한 최소한의 회원 정보만 수집한다
- 필수: 이메일, 닉네임
- 선택: 프로필 이미지

### 단일 책임 원칙
- 회원 도메인은 "누가 이 서비스를 사용하는가"에만 집중한다
- 운동 기록 등 다른 도메인의 데이터를 직접 다루지 않는다

## 3. 지원 기능
- 이메일/비밀번호 회원가입
- 이메일/비밀번호 로그인
- 로그아웃
- 회원 탈퇴 (소프트 삭제)
- 프로필 조회/수정 (닉네임, 프로필 이미지)

## 4. 미지원 기능 (현재 범위 외)
- 소셜 로그인 (추후 검토)
- 이메일 인증 (Supabase 기본 설정 따름)
- 비밀번호 재설정 UI (Supabase 기본 제공 페이지 활용)

## 5. 보안 원칙
- 회원 데이터는 본인만 조회/수정 가능하다 (RLS 적용)
- 세션 토큰은 Supabase가 관리하며 직접 핸들링하지 않는다
- 탈퇴 회원의 운동 데이터는 익명화하거나 함께 삭제한다 (정책 결정 필요)
