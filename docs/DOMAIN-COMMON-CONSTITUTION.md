# 공통 도메인 원칙

## 1. 도메인 정의 원칙
- 도메인은 비즈니스 관심사에 따라 분리한다
- 도메인 간 직접 참조는 금지하며, 필요 시 공통 타입을 통해 느슨하게 연결한다
- 각 도메인은 독립적으로 테스트 가능해야 한다

## 2. 현재 도메인 목록
| 도메인 | 설명 | 상태 |
|--------|------|------|
| Member | 회원 가입, 로그인, 프로필 관리 | 문서 작성 완료 |
| Workout | 운동 기록 CRUD, 날짜별 조회 | 문서 작성 완료 |

## 3. 새 도메인 추가 절차
1. `DOMAIN-XXX-CONSTITUTION.md` 작성 (원칙 정의)
2. `DOMAIN-XXX-STATUTE.md` 작성 (구현 규칙 정의)
3. 아키텍처 문서에 도메인 반영
4. 승인 후 구현 시작

## 4. 공통 규칙
- 모든 엔티티는 `id`, `created_at`, `updated_at` 필드를 포함한다
- `deleted_at` 소프트 삭제 방식을 기본으로 사용한다 (하드 삭제는 별도 승인)
- 날짜/시간은 UTC로 저장하고 UI에서 로컬 타임존으로 변환한다
- 사용자 입력 데이터는 서비스 레이어에서 유효성 검사를 수행한다

## 5. 공통 타입 원칙
- 공통으로 사용되는 응답 타입은 `types/common.types.ts`에 정의한다
- 페이지네이션은 cursor 기반을 기본으로 한다
