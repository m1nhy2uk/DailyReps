# 아키텍처 핵심 원칙

## 1. 기술 스택
- **프론트엔드**: Next.js (App Router)
- **백엔드/DB**: Supabase (PostgreSQL + Auth + Storage)
- **언어**: TypeScript

## 2. 레이어 원칙

### 단방향 의존성
- UI → 서비스 레이어 → DB/외부 API
- 하위 레이어가 상위 레이어를 참조하지 않는다

### 레이어 구분
| 레이어 | 책임 | 위치 |
|--------|------|------|
| UI | 화면 렌더링, 사용자 입력 | `app/`, `components/` |
| 서비스 | 비즈니스 로직, 유효성 검사 | `lib/services/` |
| 데이터 | DB 쿼리, 외부 API 호출 | `lib/repositories/` |
| 타입 | 공유 타입 정의 | `types/` |

## 3. Supabase 원칙
- 모든 DB 접근은 Supabase Client를 통해서만 수행한다
- Row Level Security(RLS)를 모든 테이블에 적용한다
- 서버 컴포넌트에서는 서버 클라이언트, 클라이언트 컴포넌트에서는 브라우저 클라이언트를 사용한다

## 4. 인증 원칙
- 인증은 Supabase Auth에 완전히 위임한다
- 보호된 라우트는 미들웨어에서 일괄 처리한다
- 클라이언트에 민감 정보(service_role key 등)를 노출하지 않는다

## 5. 상태 관리 원칙
- 서버 상태는 Supabase 실시간 구독 또는 Next.js 캐시를 활용한다
- 클라이언트 상태는 최소화하며, 필요시 React 내장 상태만 사용한다
- 전역 상태 관리 라이브러리는 도입하지 않는다 (필요 시 재검토)

## 6. 오류 처리 원칙
- 모든 서비스 함수는 `{ data, error }` 형태로 결과를 반환한다
- UI에서 에러를 무시하지 않고 사용자에게 명시적으로 표시한다
- 서버 에러 상세는 클라이언트에 노출하지 않는다
