# 운동 도메인 원칙

## 1. 도메인 목적
운동 도메인은 사용자가 날짜별로 운동 종목, 세트, 무게를 기록하고 조회하는 기능을 담당한다.

## 2. 핵심 개념

### 운동 세션 (Workout Session)
- 특정 날짜에 수행한 운동 전체를 하나의 세션으로 관리한다
- 하루에 하나의 세션만 존재한다 (날짜가 PK 역할)

### 운동 항목 (Workout Entry)
- 세션 내 하나의 운동 종목 기록
- 운동 이름(종목), 세트 수, 세트별 무게를 포함한다

### 세트 (Set)
- 운동 항목 내 개별 세트 기록
- 무게(kg), 반복 횟수(reps)를 포함한다

## 3. 데이터 구조 원칙

```
WorkoutSession (날짜)
  └── WorkoutEntry (종목) × N
        └── WorkoutSet (세트) × N
```

- 세션은 날짜로 식별한다
- 동일 날짜에 같은 종목을 두 번 기록할 수 있다 (전후 운동 등)
- 세트 순서는 저장 순서를 따른다

## 4. 비즈니스 규칙
- 사용자는 자신의 운동 기록만 조회/수정/삭제할 수 있다
- 미래 날짜에도 기록할 수 있다 (계획 기능 고려)
- 무게 단위는 kg으로 고정한다
- 무게는 소수점 한 자리까지 허용한다 (예: 52.5kg)
- 반복 횟수는 1 이상의 양의 정수이다

## 5. 운동 종목 원칙
- 종목명은 자유 텍스트로 입력한다 (별도 마스터 테이블 없음)
- 자주 사용하는 종목은 자동완성으로 제공한다 (사용자의 기존 기록 기반)
- 종목명 정규화는 수행하지 않는다 (사용자가 입력한 그대로 저장)
